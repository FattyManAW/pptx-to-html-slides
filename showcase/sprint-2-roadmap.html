新模式驗證狀態
✅ Spec-First 模式（Pipeline B）
  • D1→D2→D3 三步標準化
  • Design Stage 可重複使用
  • Dev 拿到可執行 HTML 骨架
  • --canonical gate 自動驗證
  • 全線 exit code 0
⚠️ Trunk-Based 模式（Pipeline A）
  • 直接在 main branch 開發
  • 快速迭代適合 UI 變動大的場景
  • 缺少 Design Stage 前置驗證
  • 適合單人 Dev 或小改動
  • CI Gate 仍須保留
💡 Sprint 2 任務：兩模式並存。Spec-First 用於新專案/大改動，Trunk-Based 用於小修改/熱修復。 核心原則：Spec-First 是預設，Trunk-Based 是例外。
